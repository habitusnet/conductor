## Key Concepts

### 1. Subscription-Based CLI Agents

Agents are **full CLI tools** that users authenticate via OAuth to their respective subscriptions:

| CLI Tool | Provider | Authentication |
|----------|----------|----------------|
| Claude Code | Anthropic | OAuth/Subscription |
| Codex CLI | OpenAI | OAuth/Subscription |
| Gemini CLI | Google | OAuth/Subscription |
| Cursor | Cursor | OAuth/Subscription |
| OpenCode | Community | API Key |

**Not API orchestration**: We're not programmatically calling LLM APIs. Each agent is a running CLI session with its own authentication and context.

### 2. MCP as the Coordination Layer

All CLI agents connect to Conductor via MCP (Model Context Protocol):

Agents use MCP tools to:
- Get their assigned tasks
- Claim available work
- Report progress/completion
- Check for file conflicts
- Send heartbeats

### 3. The Oversight Agent

A dedicated CLI-based agent that acts as project overseer:

**Responsibilities:**
- Monitor other agents' terminal output (via log files, tmux, or process capture)
- Detect stuck or failing agents
- Triage issues and reassign tasks
- Coordinate complex multi-agent workflows
- Escalate to API fallback when needed

**Also subscription-based**: The oversight agent can be another Claude Code or similar CLI running with elevated Conductor permissions.

### 4. API Fallback (Emergency Only)

API orchestration is reserved for:
- Agent becomes unresponsive (no heartbeat)
- CLI session crashes
- Urgent task reassignment
- Emergency intervention
- Coordination layer failures

## Agent Instruction Flow

### Reading Instructions

Agents receive instructions from multiple sources:

1. **MCP Task Queue** - Primary source via `conductor_list_tasks`
2. **Project Instructions** - CLAUDE.md, GEMINI.md files in repo
3. **Interactive Control** - Human or oversight agent sending prompts
4. **Next Action Prompts** - Self-directed based on task context

### Output Monitoring

The oversight agent monitors CLI outputs through:

1. **Log Files** - Each CLI writes to a log file
2. **Shared Sessions** - tmux/screen with read access
3. **Process Capture** - stdout/stderr redirection
4. **MCP Heartbeats** - Regular status pings with context

```bash
# Example: Running agents in monitored tmux sessions
tmux new-session -d -s claude "claude-code --project ./myapp 2>&1 | tee /var/log/conductor/claude.log"
tmux new-session -d -s gemini "gemini-cli --project ./myapp 2>&1 | tee /var/log/conductor/gemini.log"

# Oversight monitors all logs
tail -f /var/log/conductor/*.log | conductor oversight
```

## MCP Server Tools

### Task Management
| Tool | Description |
|------|-------------|
| `conductor_list_tasks` | Get available tasks for this agent |
| `conductor_get_task` | Get specific task details |
| `conductor_claim_task` | Claim a task (auction-based) |
| `conductor_start_task` | Mark task as in_progress |
| `conductor_update_task` | Update progress/status |
| `conductor_complete_task` | Mark done with results |
| `conductor_fail_task` | Report failure with error |
| `conductor_block_task` | Mark as blocked with reason |

### Coordination
| Tool | Description |
|------|-------------|
| `conductor_heartbeat` | Send keepalive with status |
| `conductor_lock_file` | Acquire file lock |
| `conductor_unlock_file` | Release file lock |
| `conductor_check_conflicts` | Check for edit conflicts |
| `conductor_list_agents` | See other active agents |

### Oversight (Elevated)
| Tool | Description |
|------|-------------|
| `conductor_reassign_task` | Move task to different agent |
| `conductor_pause_agent` | Pause an agent's queue |
| `conductor_resume_agent` | Resume agent |
| `conductor_escalate` | Trigger API fallback |
| `conductor_broadcast` | Send message to all agents |

## Security Model

### Agent Authentication
- Each CLI authenticates via OAuth to their provider
- Conductor identifies agents by their registered ID
- MCP connection carries agent identity

### Permission Levels
| Level | Capabilities |
|-------|--------------|
| Agent | Claim tasks, update own work, file locks |
| Lead Agent | Reassign tasks, priority changes |
| Oversight | All agent controls, API fallback |
| Admin | Full system access, configuration |

### Isolation
- Agents can only see/modify their assigned tasks
- File locks prevent concurrent edits
- Zone-based ownership restricts file access
